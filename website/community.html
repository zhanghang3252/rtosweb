🤝 如何贡献

MyRTOS 欢迎任何形式的贡献！无论是代码、文档、Bug 报告还是功能建议，都对项目至关重要。

贡献类型

  • 🐛 报告 Bug：在 GitHub Issues 中提交 Bug 报告，附上复现步骤和环境信息
  • 💡 功能建议：在 Discussions 或 Issues 中提出新功能建议
  • 📝 文档改进：修正文档错误、补充示例代码、翻译文档
  • 💻 代码贡献：修复 Bug、实现新功能、优化性能
  • 🔌 BSP 适配：为新的开发板添加板级支持包
  • 📦 软件包：贡献可复用的中间件/驱动软件包

代码贡献流程

# 1. Fork 仓库到你的 GitHub 账号
# 在 GitHub 页面点击右上角 "Fork" 按钮

# 2. 克隆你的 Fork
git clone https://github.com/zhanghang3252/rtosweb.git
cd myrtos

# 3. 创建功能分支
git checkout -b feature/your-feature-name

# 4. 编写代码 & 测试
# ... 编码 ...
scons  # 确保编译通过

# 5. 提交代码
git add .
git commit -m "feat(module): 描述你的改动"
# Commit 规范：feat/fix/docs/style/refactor/test/chore

# 6. 推送到你的 Fork
git push origin feature/your-feature-name

# 7. 创建 Pull Request
# 在 GitHub 页面点击 "New Pull Request"

# 8. 等待 Code Review
# 维护者会审核你的代码，可能要求修改

# 9. 合并
# 审核通过后，代码将被合并到主分支

Commit 规范

前缀 说明 示例
feat 新功能 feat(kernel): add event flag group
fix Bug 修复 fix(uart): fix rx buffer overflow
docs 文档 docs(api): update semaphore API reference
style 代码格式 style(kernel): fix code style issues
refactor 重构 refactor(scheduler): optimize priority queue
test 测试 test(mutex): add priority inheritance test
chore 构建/工具 chore(ci): update GitHub Actions workflow
